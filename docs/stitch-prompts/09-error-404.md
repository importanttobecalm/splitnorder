# 09 — Error / 404 (Hata Sayfaları)

## Stitch Promptu

```
Design the "Error" pages for Splitnorder — a friendly, on-brand experience for 404 (page not found), 500 (server error), and a generic processing error. Use the Splitnorder design system (pale icy-blue #EAF2FB dotted-grid background, white cloud-like cards with soft shadow, Plus Jakarta Sans headings, Inter body, primary #4A90E2, stem accents vocals=red, drums=orange, bass=purple, other=teal). Desktop 1440×900. Produce three sibling frames (404 / 500 / Generic).

────────────────────────
SHARED LAYOUT (all three pages)
────────────────────────
- Same top nav as the rest of the app (sticky white blurred bar with logo + nav links + avatar).
- Centered vertically and horizontally: a hero block, max-width 720px.
- Behind the hero, an ambient decoration: four very faint floating stem-shaped waveform fragments in the corners (red, orange, purple, teal, ~10% opacity) drifting slowly. Echoes the studio brand without screaming.

────────────────────────
HERO BLOCK STRUCTURE
────────────────────────
1. Big visual on top — an illustration ~280px tall:
   • A "broken" musical staff with one floating eighth-note loose and tilted, a small "?" hanging from one of the staff lines. Drawn in the same warm gradient (red→orange→amber) as the brand logo, line-art style, friendly not scary.
2. Huge numeric badge above or beside (Plus Jakarta Sans 700, 96px, gradient red→orange→amber):
     "404"  (or "500", or hidden for generic).
3. H1 — 36px 700 #1E3A5F.
4. Subhead — 17px #6B7C93, max-width 520px centered.
5. Button row, 16px gap:
   • Primary "Ana sayfaya dön" (home icon left, bg #4A90E2).
   • Ghost "Geri git" (arrow-left icon).
   • Optional ghost "Destekle iletişime geç" (lifebuoy icon).
6. Footer microcopy — 12px #A4B0C0 — error reference id "Hata kodu: #SP-2026-A47F" so the user can quote it.

────────────────────────
FRAME A — 404 (Sayfa Bulunamadı)
────────────────────────
- Big badge: "404"
- H1: "Bu stem'i bulamadık."
- Subhead: "Aradığın sayfa miks'ten çıkarılmış olabilir. Aşağıdaki düğmelerle geri dönebilirsin."
- Add a small "Belki şunu arıyordun?" row below buttons — 3 pill-shaped suggested links: "Ana sayfa", "Geçmiş projeler", "Yeni yükleme".

────────────────────────
FRAME B — 500 (Sunucu Hatası)
────────────────────────
- Big badge: "500"
- H1: "Bir şeyler ters gitti."
- Subhead: "Sunucumuz şu an miksten düştü. Ekibimiz bilgilendirildi — birkaç dakika sonra tekrar dener misin?"
- Replace primary button with "Yeniden dene" (refresh icon). Keep ghost "Ana sayfaya dön".
- Show a small live status pill beneath the buttons: a green/yellow/red dot + "Servis durumu: tüm sistemler normal" — link to status page.

────────────────────────
FRAME C — Generic Processing Error
────────────────────────
- No numeric badge.
- Illustration variant: a stem card with a small ⚠️ icon on its corner.
- H1: "Şarkını ayıramadık."
- Subhead: "Dosya bozuk olabilir veya işlem sırasında bir sorun çıktı. Başka bir dosya deneyebilirsin."
- Buttons: primary "Tekrar yükle" (upload icon) + ghost "Destekle iletişime geç".
- Beneath buttons, a small expandable "Teknik ayrıntılar ▾" link that reveals a monospace block with the technical error message — for power users only.

MOOD: human, calm, slightly playful. Errors are bummers — soften them with music-flavored metaphors ("miks'ten düşmek", "stem'i bulamadık") without being cutesy. Never alarmist.
```
